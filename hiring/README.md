# hiring — 简历分析与面试评估模块

配合项目 skill `/resume-analysis` 使用（定义见 `.claude/skills/resume-analysis/`）。

## 工作流

```
面试前：读取简历 → 多维度分析（主动性/投入度/学习能力/专业匹配/稳定性）
        → 候选人评分 → 生成 30 分钟面试计划（问题 + 追问方向）
面试后：读取面试记录 → 核对假设、修订评分 → 面试评估报告与录用建议
```

## 目录约定

每位候选人一个目录：`candidates/<YYYY-MM-DD>-<姓名>-<岗位>/`

| 文件 | 内容 |
|------|------|
| `00-岗位要求.md` | JD / 岗位核心要求 |
| `<简历原件>` | PDF/Word/图片/文本 |
| `01-简历分析报告.md` | 结构化信息、多维度分析、假设评分 |
| `02-面试计划.md` | 30 分钟议程、问题与追问链、疑点清单 |
| `03-面试记录.md` | 面试笔记/转写原文（不加工） |
| `04-面试评估报告.md` | 假设核对、修订评分、最终建议 |

## 用法

在本仓库任意会话中：
- `/resume-analysis` 后跟简历路径或直接粘贴简历文本；
- 或自然语言："分析一下这份简历""给 X 出面试题""这是 X 的面试记录，帮我评估"。

流程可从任意阶段进入——skill 会根据候选人目录中已有的编号文件判断做到了哪一步。

## 隐私

`candidates/` 下全部内容含候选人个人信息，已在 `.gitignore` 中排除，**仅本地保留，不入库不推送**。评分标准与题库（skill 目录内）不含个人信息，正常入库。
