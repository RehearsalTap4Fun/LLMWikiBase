# 输出模板

三份产出文件的模板。占位符用 `<>` 标注；表格行数按实际增减。

## 模板 A：01-简历分析报告.md

```markdown
# 简历分析报告 — <姓名>（<岗位>）

- 分析日期：<YYYY-MM-DD>
- 简历来源：<文件名/粘贴>
- 岗位要求：见 00-岗位要求.md

## 一、结构化信息

**基本情况**：<学历/年限/当前状态一句话>

**经历时间线**
| 时间 | 公司/学校 | 岗位 | 关键职责与成果 |
|------|-----------|------|----------------|

**技能与 JD 对照**
| JD 核心要求 | 满足度（满足/部分/缺失） | 简历依据 |
|-------------|--------------------------|----------|

## 二、疑点清单（面试待核实）
| # | 疑点 | 类型（断档/跳槽/含糊/矛盾） |
|---|------|------|

## 三、多维度分析

### <维度名>（每个维度一节，共五节）
- **证据**：> <简历原文引用>
- **证据强度**：强 / 中 / 弱
- **分析**：<一段话>
- **待验证假设**：<面试要确认的具体疑问>

## 四、评分（简历阶段假设分）

| 维度 | 权重 | 评分 | 一句话理由 | 证据强度 |
|------|------|------|-----------|----------|
| 专业能力与岗位匹配 | 30% | | | |
| 主动性 | 20% | | | |
| 投入度 | 20% | | | |
| 学习能力 | 20% | | | |
| 稳定性与职业连贯性 | 10% | | | |
| **加权总分** | | **<x.x>** | 档位：<强烈推荐/推荐/待定/不推荐> | |

- 单项风险：<无，或列出 ≤2 分的维度及说明>
- 面试优先验证：<维度/疑点排序>

> 本评分为简历阶段假设，以面试后修订版（04-面试评估报告）为准。
```

## 模板 B：02-面试计划.md

```markdown
# 面试计划 — <姓名>（<岗位>，30 分钟）

面试重点：<本场要验证的 2–3 个维度/疑点，一句话>

## 议程
| 时段 | 内容 |
|------|------|
| 0–3 min | 开场 + 自我介绍（限 2 分钟） |
| 3–11 min | 核心项目深挖：<项目名> |
| 11–24 min | 维度验证问题 ×<N> |
| 24–27 min | 快问快答：疑点核实 |
| 27–30 min | 候选人提问 + 收尾 |

## 核心项目深挖（3–11 min）
**切入**：<针对简历中该项目的开场问题>
**必挖细节**：<数字来源/技术决策/最难的坑 等 2–3 个点>

## 维度验证问题（11–24 min）

### Q1（考察：<维度>）
**问**：<绑定简历的行为面试问题>
- 追问 1（背景）：<>
- 追问 2（行为）：<>
- 追问 3（结果/反思）：<>
- 防背稿：<只有真做过才答得出的细节点>
- 评估要点：好答案 <特征>；差答案 <特征>

（Q2、Q3 同结构）

## 快问快答（24–27 min）
- [ ] <疑点 1 的中性问法>
- [ ] <疑点 2>
- [ ] 求职动机与稳定性确认

## 观察项
- 自我介绍结构性；候选人提问质量（主动性/投入度信号）
```

## 模板 C：04-面试评估报告.md

```markdown
# 面试评估报告 — <姓名>（<岗位>）

- 面试日期：<YYYY-MM-DD>；记录来源：03-面试记录.md

## 一、假设核对

| 简历阶段假设 | 结论（证实/证伪/仍不确定） | 依据（面试原话摘引） |
|--------------|---------------------------|----------------------|

## 二、评分修订

| 维度 | 简历分 | 面试后分 | 变化原因 |
|------|--------|----------|----------|
| 专业能力与岗位匹配 | | | |
| 主动性 | | | |
| 投入度 | | | |
| 学习能力 | | | |
| 稳定性与职业连贯性 | | | |
| **加权总分** | <x.x> | **<x.x>** | 档位：<> |

## 三、亮点与红旗
- 亮点：<引用具体回答>
- 软红旗：<表达/准备类问题>
- 硬红旗：<诚信/造假嫌疑；无则写"无">

## 四、结论与建议
- **结论**：强烈推荐 / 推荐 / 待定 / 不推荐
- **理由**（3 点以内）：
- **后续动作**：<进入下轮（下轮验证什么）/ 加面 / 背调重点 / 婉拒>
- 仍未验证的疑问：<留给下轮或背调>
```
